# 공정성 검증 가이드

## 공정성 보장 시스템

BlockPick은 블록체인 기술과 암호학을 활용하여 **100% 투명하고 검증 가능한** 게임 환경을 제공합니다.

### 3가지 핵심 원칙

```
1. 투명성 (Transparency)
   모든 게임 과정이 블록체인에 공개적으로 기록

2. 불변성 (Immutability)
   기록된 데이터는 누구도 수정할 수 없음

3. 검증 가능성 (Verifiability)
   누구나 언제든지 직접 검증 가능
```

## 블록체인 기반 투명성

### Polygon 블록체인

```
네트워크: Ethereum Mainnet
컨트랙트 주소: 0x1234... (예시)

특징:
✅ 전 세계 수천 개 노드가 검증
✅ 조작 불가능
✅ 영구 보존
✅ 공개 열람 가능
```

### 기록되는 정보

```
게임 생성:
- 게임 ID
- 격자 크기
- 참가비
- 생성 시간

참여자 Commit:
- 참가자 주소 (익명화)
- Commit 값 (암호화된 선택)
- Commit 시간
- Transaction Hash

참여자 Reveal:
- 참가자 주소
- 실제 선택 블록
- Reveal 시간
- Transaction Hash

당첨자 결정:
- 당첨자 주소 (익명화)
- 당첨 블록
- 당첨 금액/상품
- 결정 시간
- Transaction Hash
```

## 검증 방법

### 방법 1: BlockPick 웹사이트에서 확인

#### 1단계: 게임 이력 접속

```
로그인 후:
마이페이지 > 게임 이력
   ↓
확인하고 싶은 게임 클릭
   ↓
게임 상세 정보 확인
```

#### 2단계: 검증 버튼 클릭

```
게임 상세 페이지에서:
"블록체인 검증" 버튼 클릭
   ↓
새 창으로 PolygonScan 열림
   ↓
블록체인 기록 확인
```

```

│ [블록체인 검증하기] 🔗       │
└─────────────────────────────┘
```

### 방법 3: 스마트 컨트랙트 소스 코드 확인

```
PolygonScan > Contract 탭
   ↓

- 스마트 컨트랙트 소스 확인

- 로직 분석
```

**스마트 컨트랙트 함수 예시**
```solidity
// 게임 정보 조회
function getGameInfo(uint256 gameId)
  public view returns (
    uint8 gridSize,
    uint256 entryFee,
    uint256 participants,
    address winner
  );

// 참가자의 Commit 확인
function getCommit(uint256 gameId, address player)
  public view returns (bytes32 commitment);

// 참가자의 Reveal 확인
function getReveal(uint256 gameId, address player)
  public view returns (uint8 blockChoice);
```

## 단계별 검증 가이드

### Commit 단계 검증

#### 확인 사항

```
1. Commit 값이 블록체인에 기록되었는가?
   ✅ PolygonScan에서 TX Hash 확인

2. Commit 시간이 정확한가?
   ✅ Block Timestamp 확인

3. Commit 값이 암호화되었는가?
   ✅ Input Data가 해시 값인지 확인
```

**예시**
```
Commit Transaction:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
TX Hash: 0xabcd1234...
Block: 18,234,567
Timestamp: 2024-02-15 14:23:17
Function: commitChoice(bytes32)
Input: 0x5f3a... ← 암호화된 선택
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 블록체인에 정상 기록됨
✅ 다른 사람이 선택을 알 수 없음
```

### Reveal 단계 검증

#### 확인 사항

```
1. Reveal 값이 Commit 값과 일치하는가?
   ✅ Hash(블록 번호 + Nonce) = Commit 값?

2. Reveal이 정해진 시간 내에 이루어졌는가?
   ✅ Reveal 타임스탬프 확인

3. 모든 참가자가 Reveal했는가?
   ✅ Reveal 이벤트 개수 = Commit 개수?
```

**예시**
```
Reveal Transaction:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
TX Hash: 0xefgh5678...
Block: 18,234,890
Timestamp: 2024-02-15 14:30:02
Function: revealChoice(uint8, string)
Input:
  blockChoice: 7
  nonce: "a7f3k2m9"

검증:
SHA256(7 + "a7f3k2m9") = 0x5f3a...
Commit 값: 0x5f3a...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Commit 값과 일치!
✅ 유효한 Reveal
```

### 당첨자 결정 검증

#### 확인 사항

```
1. 블록별 선택 인원이 정확한가?
   ✅ 모든 Reveal 값을 집계
   ✅ 각 블록의 선택자 수 확인

2. 최소 선택 블록이 정확한가?
   ✅ 가장 적은 선택 인원 블록 확인

3. 당첨자가 가장 빨리 Commit했는가?
   ✅ 최소 블록 선택자들의 Commit 시간 비교
   ✅ 가장 이른 타임스탬프 확인
```

**예시**
```
게임 #1234 결과:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
블록별 선택 인원:
블록 1: 3명
블록 2: 5명
블록 3: 2명
...
블록 7: 1명 ← 최소!

블록 7 선택자:
참가자 A (0x9876...)
Commit 시간: 2024-02-15 14:23:17

당첨자: 참가자 A
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

검증:
✅ 블록 7이 최소 선택 (1명)
✅ 참가자 A가 유일한 선택자
✅ 정당한 당첨
```

## 의심 사례 검증

### 사례 1: 중복 참여 의심

```
검증 방법:

   ↓

   ↓

require(commits[msg.sender] == 0)
   ↓
✅ 중복 참여 불가능 (코드로 차단)
```

### 사례 2: 선택 변경 의심

```
검증 방법:
Commit TX와 Reveal TX 비교
   ↓
Hash(Reveal 값) = Commit 값?
   ↓
불일치 시:
→ 스마트 컨트랙트가 자동 거부
→ Reveal 실패 이벤트 발생
   ↓
✅ 선택 변경 불가능
```

### 사례 3: 당첨자 조작 의심

```
검증 방법:
모든 Reveal 값을 직접 집계
   ↓
최소 블록 찾기
   ↓
해당 블록 선택자의 Commit 시간 비교
   ↓
가장 빠른 사람 = 당첨자?
   ↓
✅ 일치하면 정당한 당첨
❌ 불일치하면 신고
```

## 자주 묻는 질문

### Q1. 블록체인 검증이 어렵게 느껴져요.
**A.** BlockPick 웹사이트에서 "블록체인 검증" 버튼만 클릭하면 자동으로 PolygonScan으로 연결됩니다. 기술 지식이 없어도 확인 가능합니다.

### Q2. 관리자가 결과를 조작할 수 있나요?
**A.** 불가능합니다. 스마트 컨트랙트가 자동으로 실행되며, 배포 후에는 누구도 수정할 수 없습니다.

### Q3. PolygonScan은 무엇인가요?
**A.** Polygon 블록체인의 모든 거래를 볼 수 있는 공개 탐색기입니다. Google과 같은 검색 엔진과 유사합니다.

### Q4. 검증 결과가 이상한데 어디에 신고하나요?
**A.** report@blockpick.io로 상세한 내용과 함께 신고해주세요. 24시간 내에 확인하겠습니다.

### Q5. 스마트 컨트랙트 소스 코드를 볼 수 있나요?
**A.** 네, PolygonScan의 Contract 탭에서 전체 소스 코드를 확인할 수 있으며, GitHub에서도 공개되어 있습니다.

### Q6. 왜 블록체인을 사용하나요?
**A.** 투명성과 불변성을 보장하기 위해서입니다. 중앙 서버는 관리자가 조작할 수 있지만, 블록체인은 조작이 불가능합니다.

### Q7. 다른 참가자의 정보도 볼 수 있나요?
**A.** 블록체인에는 지갑 주소만 기록되어 개인 식별 정보는 보호됩니다. 완전히 익명화되어 있습니다.

### Q8. 검증은 필수인가요?
**A.** 필수는 아니지만, 신뢰를 위해 권장합니다. 누구나 언제든지 검증할 수 있다는 것이 BlockPick의 핵심 가치입니다.
